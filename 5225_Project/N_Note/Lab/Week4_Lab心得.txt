=== 仅Master机 ===
$ sudo kubeadm init --pod-network-cidr=10.244.0.0/16 --cri-socket=unix:///var/run/cri-dockerd.sock  --apiserver-advertise-address=<master-private-IP-address>

 kubeadm init用来初始化kubenetes
 --pod-network-cidr=10.244.0.0/16 设置pod内部网络的地址（默认）
 --cri-socket=unix:///var/run/cri-dockerd.sock 选择容器运行时通信地址为 cri-dockerd,因为kubernetes现在不支持docker了
 --apiserver-advertise-address=<master-private-IP-address> 选择Master对外ip地址（master ip)，为了内部可以互相通信，其实也可以选择public ip(虽然不推荐）


$ mkdir -p $HOME/.kube 创建.kube文件夹（默认Kubectl配置目录）
$ sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config 把kubernetes里面admin的配置文件复制到Kubectl默认目录位置，改名为config
$ sudo chown $(id -u):$(id -g) $HOME/.kube/config 更改config的所有者（和权限）,以前拥有者是root，所以普通用户无法访问
 - $(id -u)当前用户UID
 - $(id -g)当前用户组GID

=== Task3 ====

Namespace: Kubernetes 的"虚拟文件夹"，用来把资源逻辑分组和隔离，让一个集群像多个独立集群一样用。

$ kubectl get namespace 查看当下namespace
$ kubectl create namespace <your-name> 创建新的namespace 

$ kubectl get nodes,deployments,pods --namespace=<your-namespace> -o wide
在你的namespace下查看节点，Deployment和Pod的信息
Deployment控制Pods，下发任务，开启/关闭pod等
Pod真正做任务
-o wide: option wide，宽表输出(更多信息)

$ kubectl create deployment --image nginx my-nginx-server --namespace=<your-namespace>
kubectl create deployment: 让kubectl创建deployment
--image nginx运行nginx，其实是dockerhub的image(因为用了cri-dockerd)
my-nginx-server deployment名
--namespace=<your-namespace> 部署到指定namespace

也可以创建yml文件:

apiVersion: apps/v1 # API 版本，apps/v1 是 Deployment 的稳定版本
kind: Deployment # 资源类型，告诉 Kubernetes 创建 Deployment 对象
metadata: # 元数据部分
  name: nginx-deployment # Deployment 名称，集群里唯一标识
spec: # 期望状态规格
  selector: # 标签选择器，Deployment 管理所有带有 app=nginx 标签的 Pod
    matchLabels:
      app: nginx
  replicas: 2 # 副本数，保持 2 个 Pod 运行（挂了自动补）
  template: # Pod 模板，Deployment 创建 Pod 的蓝图
    metadata:
      labels: # Pod 标签，必须和上面 selector 匹配
        app: nginx
    spec:
      containers: # 容器定义列表
      - name: nginx # 容器名称
        image: nginx:latest # Docker 镜像（从 Docker Hub 拉取）
        ports: # 容器监听端口，nginx 默认 80 端口
        - containerPort: 80


$ kubectl apply -f my-deployment.yml --namespace=<your-namespace>
声明式部署 YAML 文件中的资源到指定 Namespace。

$ kubectl logs <pod-name> --namespace=<your-namespace>
查看具体的一个pod

$ kubectl exec -it <pod-name>  --namespace=<your-namespace> -- /bin/bash
kubectl exec: 在pod容器内执行命令
-it: -i,--stdin	保持标准输入打开,允许你输入命令（键盘敲 ls、cat 等）,-t,--tty，分配伪终端，提供终端界面
<pod-name>: pod名
--namespace=<your-namespace>: namespace名
--: 分隔符
/bin/bash: 要执行的命令

$ kubectl scale deployment nginx-deployment --replicas=4 --namespace=<your-namespace>
kubectl scale: 调整副本数量
deployment nginx-deployment: 目标是nginx-deployment
--replicas=4 新副本数量
--namespace=<your-namespace> namespace名

创建一个叫my-servce.yml的文件
apiVersion: v1 
kind: Service
metadata:
  name: my-service # Service name, used for internal DNS
spec:
  type: NodePort # Exposes a fixed port on every node
  selector:
    app: nginx # Routes traffic to Pods with label app=nginx
  ports:
    - protocol: TCP # Protocol type
      port: 8080 # Service port, internal access via my-service:8080
      targetPort: 80 # Container's listening port (nginx default 80)
      nodePort: 30003 # External access via NodeIP:30003

$ kubectl apply -f my-service.yml --namespace=<your-namespace>
